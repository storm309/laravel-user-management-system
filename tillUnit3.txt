// Unit 3: Resourceful Routing in Laravel
Resourceful routing in Laravel provides a convenient way to create routes that handle typical CRUD (Create, Read, Update, Delete) operations for a resource.
By using the `Route::resource` method, you can automatically generate all the necessary routes for a controller that follows the standard naming conventions.
Hereâ€™s how you can set up resourceful routing for a `ProductController` in Laravel:
1. First, ensure you have a `ProductController` created. If you haven't created it yet, you can do so using the following command:
   ```bash
   php artisan make:controller ProductController --resource
   ```
   This command will create a controller with methods for all the standard CRUD operations: `index`, `create`, `store`, `show`, `edit`, `update`, and `destroy`.
2. Next, you need to define the resourceful routes in your `routes/web.php` file. You can do this by adding the following line:
   ```php
   use App\Http\Controllers\ProductController;
    Route::resource('products', ProductController::class);
    ```
    This will generate the following routes:
| HTTP Method | URI                | Action       | Route Name         |
|-------------|--------------------|--------------|--------------------|
| GET         | /products          | index        | products.index     |
| GET         | /products/create   | create       | products.create    |
| POST        | /products          | store        | products.store     |
| GET         | /products/{id}     | show         | products.show      |
| GET         | /products/{id}/edit| edit         | products.edit      |
| PUT/PATCH   | /products/{id}     | update       | products.update    |
| DELETE      | /products/{id}     | destroy      | products.destroy   |
